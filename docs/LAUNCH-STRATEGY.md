## Phase Overview

| Phase | Dates | Focus | Users |
|-------|-------|-------|-------|
| **Test** | Feb 6-28 | Black Meat Market + feedback | ~50 testers |
| **Build** | Mar 1-31 | Iterate based on feedback | Internal |
| **Soft Launch** | Apr 1-30 | Partners + key influencers | ~200 invited |
| **Marketing** | May 1-31 | Word of mouth campaign | Building buzz |
| **Launch** | June (pre-Pride) | Gala event + public launch | Open registration |

---

## February: Test Phase

### Black Meat Market Away Weekend (Feb 6-8)

**Goal**: Real-world test with friendly audience

**What we need**:
- Working profiles with photos
- Browse attendees
- Like/Match system
- Basic messaging
- Event page with schedule
- Icebreaker prompts

**What we're testing**:
- Core user flows
- Technical stability
- Feature gaps
- Community dynamics

### Post-Weekend (Feb 10-28)

**Activities**:
- Collect feedback (surveys, 1:1 conversations)
- Identify critical bugs
- Understand what's missing
- Document learnings

**Deliverable**: Test report with prioritized improvements

---

## March: Build Phase

### Iterate Based on Real Feedback

| Week | Focus |
|------|-------|
| 1 | Bug fixes from BMM weekend |
| 2 | Top 3 user-requested features |
| 3 | Safety Check-In system |
| 4 | Mediation flow + Community agreements |

**No new features without user evidence**. Build what testers actually asked for.

---

## April: Soft Launch

### Invite-Only Release

**Who**:
- Partner organizations (UK Black Pride, GMFA, NAZ, etc.)
- Key community influencers (10-15 people)
- BMM testers who want to continue
- Hand-picked early adopters

**Target**: ~200 active users

**Goals**:
- Validate product-market fit
- Build core community culture
- Train community stewards for mediation
- Stress test at small scale

### Cohort Alpha Kickoff

- Community agreements vote
- Welcome event (virtual or in-person)
- Set the cultural tone before public launch

---

## May: Marketing Phase

### Slow-Burn Word of Mouth

**Strategy**: Let early users be the messengers

| Tactic | Description |
|--------|-------------|
| **Testimonials** | Video/written stories from soft launch users |
| **Partner promotion** | Partner orgs share with their communities |
| **Social teases** | "Something's coming for Pride..." |
| **Influencer posts** | Authentic endorsements, not ads |
| **Waitlist** | Build anticipation with registration |
| **Community events** | Pre-launch mixers (soft launch users + friends) |

**Not doing**:
- Paid ads
- Aggressive growth hacking
- Influencer sponsorships
- Mass email blasts

**Tone**: Exclusive, community-first, word spreading naturally

## June: Launch

### Pre-Pride Gala Event

**Timing**: Week before UK Black Pride (typically late June)

**Format**:
- Evening launch party
- Venue in London
- Mix of celebration + app onboarding
- Live demonstrations
- Community agreements ceremony
- Photo booth (content for socials)

**Invites**:
- All soft launch users
- Waitlist (first come basis)
- Partner organizations
- Press/media (selective)
- Community figures

### Public Launch

- App available to all
- Cohort system begins properly
- Registration opens
- Marketing ramps up

### Pride Weekend

- DOWN presence at UK Black Pride
- Activation/booth if possible
- In-app Pride event features
- Community meetup

## Resource Implications

### Sustainable Pace

| Phase | Intensity | Notes |
|-------|-----------|-------|
| Dec-Jan | Medium | Core build, no rush |
| Feb | Low-Medium | Test, observe, learn |
| Mar | Medium | Respond to feedback |
| Apr | Low | Let users use it |
| May | Low (dev) / Medium (marketing) | Focus shifts |
| Jun | High | Launch push |

### What This Enables

- Proper testing with real users
- Time to respond to feedback
- No crunch or burnout
- Community culture established before scale
- Marketing grounded in real user stories
- Launch event with momentum behind it
